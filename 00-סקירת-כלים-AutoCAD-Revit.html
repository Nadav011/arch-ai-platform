פרומפטים לדוגמה

מה אפשר להגיד ל-Claude ומה קורה בפועל

🏗️ AutoCAD — יצירת שרטוט
"צייר מגרש מלבני של 20 מטר על 12 מטר בשכבה SITE-BOUNDARY. הוסף בניין של 12×8 מ' במרכז עם offset מינימום של 3 מ'. תייג את כל הממדים בעברית ותוסיף חיצי צפון."
📋 AutoCAD — Batch Processing
"עבור על כל קבצי ה-DWG בתיקייה /projects/floors ותוסיף לכל אחד לוח כותרת עם הלוגו של המשרד, שם הפרויקט, תאריך היום, ומספר גיליון רץ."
🏢 Revit — ניתוח מודל
"בדוק את המודל ותגיד לי: כמה מ"ר שטח בנייה יש בכל קומה, אילו חדרים חסרים חלון (בלי opening בקיר חיצוני), והכן לי טבלת חומרים לכל הקירות בפרויקט."
🔍 Revit — QC אוטומטי
"הרץ בדיקת איכות מלאה על המודל: - פרמטרים חסרים בחדרים (שם, מספר, שימוש) - דלתות ללא Room Separation - קירות שלא מוגדרת להם Function - הצג דוח מסכם עם רשימת תיקונים לפי עדיפות"
📄 Revit — הפקת מסמכים
"צור Sheet Set לכל הקומות בפרויקט. לכל קומה: תוכנית רצפה, תוכנית תקרה, 4 חזיתות. שמור את ה-Viewport Scales כ-1:100 ותמקם אוטומטית."
🤖 AutoLISP — כתיבת קוד
"כתוב AutoLISP routine שסורק את כל ה-Polylines בשרטוט, מחשב את שטח כל אחת, ומוסיף Mtext עם השטח במ"ר במרכז כל Polyline. תסביר את הקוד בעברית."

כיצד מתחילים —
4 שלבים למשרד

תוכנית יישום מעשית ל-60 יום ראשונים

שבוע 1–2 — שלב א'
התקנה והדגמה
התקנת Claude Desktop + הגדרת MCP Server אחד (NonicaTab לRevit, או AutoCAD MCP Pro לAutoCAD). הדגמה חיה לצוות: 3–5 פרומפטים ראשונים, הצגת תוצאות, קבלת פידבק. פלט: הצוות ראה את זה עובד
שבוע 3–4 — שלב ב'
אוטומציה ראשונה
זיהוי 3 תהליכים חוזרים שגוזלים הכי הרבה זמן — לוחות כותרת, שכבות, QC. כתיבת prompt templates לכל אחד. הגדרת library של פרומפטים ל-Claude. פלט: חיסכון של 2–4 שעות שבועיות
שבוע 5–7 — שלב ג'
שילוב בזרימת עבודה
אינטגרציה מלאה: QC אוטומטי לפני כל הגשה, Batch Processing לעדכוני פרויקט, תבניות Revit עם פרמטרים סטנדרטיים, AutoLISP Routines לחישובים שכיחים. פלט: זרימת עבודה חדשה מוגדרת
שבוע 8–10 — שלב ד'
הכשרה ומדידה
הכשרת כל הצוות, מדידת זמנים לפני/אחרי, בניית Prompt Library פנימי של המשרד, שקילת שדרוג ל-Revit 2027 לMCP מובנה, בחינת Claude API לאוטומציות מותאמות. פלט: ROI מוכח, תוכנית להמשך
