שאלות נפוצות

ההתנגדויות האמיתיות

אלה השאלות שכל CTO ומנהל שואל לפני שהוא מחליט. תשובות ישירות, בלי שיווק.

Claude מחובר ל-Revit דרך MCP (Model Context Protocol) — זה לא chatbot שאתה מסביר לו מה יש במודל. הוא קורא את הפקודות ישירות, מריץ queries, ומחזיר תוצאות מבוססות מידע אמיתי. ה-87 שניות של QC הן לא "תיאור" — הן סריקה בפועל של 340 אלמנטים.
הממצאים תמיד מגיעים עם reference לאלמנט הספציפי ב-Revit — Element ID, רמה, מיקום. האדריכל יכול לוודא ידנית בלחיצה. בדיקות QC של Claude מיועדות להיות "רשת ביטחון" — לא להחליף שיפוט מקצועי. מנסיון: שגיאות false-positive מתרחשות ב-<3% מהממצאים.
ניתן לפרוס את המערכת ב-3 מודלים: (1) Anthropic API — נתונים לא נשמרים לאחר ה-session, לא משמשים לאימון. (2) AWS Bedrock ב-region ישראל/EU — נתונים לא יוצאים מה-region. (3) On-premise עם Claude models מורשים. עבור מרבית המשרדים, Anthropic API + עיבוד לוקאלי של קבצי Revit מספיקים.
